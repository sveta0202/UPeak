# Подключение Google Sheets к форме «Участвовать»

Эта инструкция описывает, как сохранить заявки со страницы `participate.html` в Google Таблицу через Google Apps Script Web App.

## 1. Создайте Google Таблицу

1. Откройте [https://sheets.google.com](https://sheets.google.com) и нажмите «Создать пустую таблицу».
2. Назовите таблицу, например, **Upeak — Заявки**.
3. Переименуйте первый лист в `Participants` (или оставьте `Sheet1` — мы поменяем имя через скрипт).

## 2. Структура колонок

Скрипт сам создаёт строку заголовков при первом запуске. Колонки будут такие:

| # | Колонка                 | Описание                                                              |
|---|-------------------------|------------------------------------------------------------------------|
| A | Timestamp               | Время записи на сервере (Apps Script)                                  |
| B | Name                    | Имя пользователя (обязательное поле)                                   |
| C | Phone                   | Телефон в исходном виде, как ввёл пользователь                         |
| D | Telegram                | Никнейм в формате `@username` (пусто, если не указан)                  |
| E | Language                | Язык интерфейса при отправке (`ru` или `en`)                           |
| F | Source Page             | Полный URL страницы, с которой отправлена заявка                        |
| G | User Agent              | Браузер пользователя                                                   |
| H | Submitted At (client)   | Время на устройстве пользователя (ISO 8601)                            |
| I | Status                  | Статус заявки. По умолчанию `new`, можно менять вручную                 |

> Если хочется создать заголовки заранее — скопируйте названия из таблицы выше в первую строку листа `Participants`.

## 3. Откройте Apps Script и вставьте код

1. В таблице меню → **Расширения → Apps Script**.
2. В открывшемся проекте удалите содержимое файла `Код.gs` (`Code.gs`).
3. Скопируйте содержимое файла `google-apps-script/Code.gs` из этого репозитория и вставьте в редактор.
4. Сохраните проект (`Ctrl+S` / `Cmd+S`), назовите его, например, **Upeak — Participants**.

### Настройка переменных в скрипте

В начале файла есть две переменные:

```javascript
var SPREADSHEET_ID = "";
var SHEET_NAME = "Participants";
```

* Если скрипт открыт **из меню таблицы** (привязан к ней) — `SPREADSHEET_ID` можно оставить пустым: будет использована активная таблица.
* Если скрипт **standalone** (отдельный проект) — вставьте `SPREADSHEET_ID` из URL вашей таблицы (часть между `/d/` и `/edit`).
* `SHEET_NAME` — имя листа. Если такого листа нет, скрипт создаст его автоматически.

### Первый запуск (даст разрешения и создаст заголовки)

1. В редакторе скриптов выберите функцию `setup` в выпадающем списке.
2. Нажмите **Run** (▶). Google попросит выдать разрешения — согласитесь.
3. Откройте таблицу: на листе `Participants` появятся заголовки.

## 4. Деплой как Web App

1. В Apps Script → **Deploy → New deployment**.
2. Выберите тип **Web app**.
3. Заполните поля:
   * **Description**: `Upeak participants endpoint v1`
   * **Execute as**: **Me** (`<ваш email>`)
   * **Who has access**: **Anyone** *(важно — иначе фронтенд получит 401/302)*
4. Нажмите **Deploy** и при необходимости пройдите аутентификацию.
5. Скопируйте полученный **Web app URL** — он выглядит как:
   `https://script.google.com/macros/s/AKfyc.../exec`

## 5. Подключите URL к форме

Откройте файл `public/participate.html` и найдите атрибут `data-script-url` у формы:

```html
<form id="participateForm"
      data-script-url="PASTE_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE"
      ...>
```

Замените плейсхолдер на ваш URL:

```html
<form id="participateForm"
      data-script-url="https://script.google.com/macros/s/AKfyc.../exec"
      ...>
```

Сохраните файл и задеплойте сайт (Railway/любой статик-хостинг).

## 6. Проверка

1. Откройте `/participate.html` в браузере.
2. Заполните форму (Имя, Телефон; Telegram — необязательно).
3. Нажмите «Отправить заявку» и убедитесь, что появляется зелёный баннер успеха.
4. Откройте Google Таблицу — должна появиться новая строка.
5. Дополнительно: откройте Web app URL в браузере (GET) — должно вернуться `{"ok":true,...}`.

## 7. Обновление кода скрипта

После каждого изменения `Code.gs` нужно либо:
* нажать **Deploy → Manage deployments → Edit (карандаш) → New version → Deploy**, либо
* создать новый деплой и заменить URL в `participate.html`.

Самый простой путь — **Manage deployments → Edit → Version: New version → Deploy**: URL не меняется.

## Технические заметки

* Фронтенд отправляет POST с `Content-Type: text/plain;charset=utf-8`. Это «простой» CORS-запрос, поэтому Apps Script отдаёт ответ без preflight-проблем. JSON находится в теле запроса (`e.postData.contents`).
* Скрипт также понимает `application/x-www-form-urlencoded` на случай резервной отправки.
* Серверная валидация: `name` обязателен, `phone` должен содержать ≥ 7 цифр. При нарушении возвращается `{"ok":false,"error":"..."}`.
* Telegram нормализуется на сервере: ведущий `@` добавляется автоматически, если был отправлен без него.
