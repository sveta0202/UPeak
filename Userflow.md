# **User flow**

1. Пользоватлель попадает на страницу лендинга
2. Переходит на страницу регистрации
3. Регистрируется и получает свой ID
4. Организатор исследования связывается с пользователем и отправляет ему даты, в которые будет проходить исследование, инструкцию по работе с прототипом, подробности эксперимента. 
5. В назначенные даты пользователь работает со страницей прототипа (с планером):\
*Алгоритм работы написан, опираясь на функционал страницы во время 2 недели исследования.*\
**5.1.** Утром заполняет блок "Утрений чек-ин состояния", стразу же получает оценку состояния.\
**5.2** В течении дня работает со вторым блоком "Задачи на день", добавляет задачу и нажимает кнопку "Распределить по состоянию", после чего следует порядку выполнения задач, который отображается в столбце "Рекомендованный слот".\
**5.3** Алгоритм также может перенести некоторые задачи в блок "Запланированные на завтра", эти задачи добавлятся во второй блок на следующий день.\
**5.4** Вечером, после выполнения всех задач, пользователь заполняет данные о своём состоянии в блок "Вечерний итог дня" и нажимает кнопку "Закрыть день".
6. По завершении эксперимента организваторы формируют статистику по каждому пользователю и отправляют ее ему для ознакомления.
